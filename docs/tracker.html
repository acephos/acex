Philosophy · four pillars

* Extensibility

Traits, adapters, registries. Editor bridge. Transport trait. Palette as registry (upcoming).

* Platform-agnostic core

Types + model compile everywhere. IO isolated. Unix-first delivery, not Unix-only design.

* Performant agentic load

Event reduce hot path. Non-blocking wait UX. Snapshot replace on reconnect only.

* Observable parallel work

Honest board. Peeks = cache. Herdr + FS = authority. Scannable states over nested chrome.

  │                 events.subscribe (long-lived)

  └── acex-editor ──▶ zed
  └── handoff ──▶ herdr attach
  • Bootstrap: connect → ping → snapshot → subscribe → draw
  • Steady: events → reduce → draw; UI → request(id) → model
  • Reconnect: drop sub → backoff → snapshot replace → resubscribe
  • Peek buffers are never authority.
Demo script (G1 / G1.5)

Prereqs: herdr, zed on PATH; repo ~/proj; binary acex.

  1. Stop any default session if needed
  2. Launch acex from project → auto-connect
  3. Empty board → CTA visible
  4. Palette → Start agent
  5. Board shows agent; state updates
  6. Select → peek recent output
  7. Send text → peek refresh
  8. Wait until done/blocked (timeout OK)
  9. Open in Zed on workspace cwd
  10. Attach in Herdr → detach; acex still live
  11. Kill server → reconnect → board restored
  12. (1.5) Worktree create → list → open in Zed
  13. (1.5) Layout preset → no live PTY warning → new tab
  14. (1.5) Diff via zed --diff
  15. Quit acex; leave server running
